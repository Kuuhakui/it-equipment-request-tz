# 📋 Техническая документация: Система управления ИТ-оборудованием

Добро пожаловать в репозиторий с артефактами системного анализа. Здесь собраны решения по автоматизации процессов компании «X», выполненные в рамках учебного проекта.

## 🚀 О проекте
В данном репозитории представлена документация для системы обработки заявок на ИТ-оборудование. Проект охватывает путь от возникновения потребности пользователя до реализации API-интерфейсов.

---

## 📂 Структура проекта

### 1. Моделирование бизнес-процессов (BPMN)
* **Инструмент:** Camunda Modeler.
* **Описание:** Отрисован процесс "As-Is" (Как есть) для выдачи оборудования с учетом взаимодействия между техподдержкой, снабжением и внешними поставщиками.


### 2. Спецификация требований (User Story & Use Case)
* **Инструмент:** MS Visio, Obsidian.
* **Описание:** Разработана функциональность публикации товара на маркетплейсе. 
* **Содержание:**
    * **User Story:** Верхнеуровневое описание ценности для Продавца.
    * **Use Case:** Детальный сценарий взаимодействия с системой, включая альтернативные потоки.
    * **UML Diagram:** Диаграмма вариантов использования.

### 3. Проектирование REST API и алгоритмов
* **Инструмент:** Obsidian (Mermaid), Markdown.
* **Описание:** Проектирование метода регистрации пользователя.
* **Содержание:**
    * Описание эндпоинта `POST /register`.
    * Sequence Diagram взаимодействия Frontend <-> Backend <-> DB.
    * Блок-схема (Activity Diagram) алгоритма валидации и создания пользователя на бэкенде.

## 🛠 Технологический стек
* **Нотации:** BPMN 2.0, UML (Use Case, Sequence, Activity).
* **Инструменты:** Camunda Modeler, MS Visio, Obsidian, Git.
